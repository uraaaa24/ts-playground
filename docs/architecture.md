# アーキテクチャ/ディレクトリ構成

## 目標
作品に関係するコードと共通基盤を分離し、作品実装に集中できる状態を維持する。

## ディレクトリ構成
```
src/
  artworks/          # 作品ファイル（1作品1ファイル）
    aurora-pulse.ts
    particle.ts
  core/              # 共通基盤（作品に共通の型・パレット・ローダー）
    p5-loader.ts
    palettes.ts
    types.ts
  registry.ts        # 作品一覧の登録
  main.ts            # ルーティングと描画の入口
  style.css          # ホーム画面と作品画面の見た目
  index.ts           # 実行案内のログ
```

## モジュールの役割
### `src/artworks/*`
作品を 1 ファイルずつ実装する場所。`mount(container)` を実装し、描画開始と終了処理を完結させる。

### `src/core/*`
作品間の共通処理を集約する場所。パレット、型定義、p5 のロードなどを集中管理する。

### `src/registry.ts`
作品一覧の登録と、`slug` から作品を引く検索関数を保持する。

### `src/main.ts`
URL からホーム/作品の描画を切り替えるルーティングと、クリーンアップ処理の実行を担当する。
